# Security Policy

Scanly is intended to be maintained as a public open-source repository, so security issues should be handled carefully.

## Reporting a vulnerability

If you find a security issue:

- do not post exploit details in a public issue
- include the affected version or commit, device details, and clear reproduction steps
- describe any data exposure, permission impact, or crash behavior you observed

## Disclosure guidance

Use the repository's private security disclosure path if one is available.
If no private path exists yet, open a minimal public issue that says a security report needs maintainer attention, and avoid including sensitive details in the issue body.

## What to include in a report

- affected Scanly version or commit
- Android version and device model
- exact steps to reproduce
- whether the issue is reproducible on a clean install
- any logs or screenshots that do not contain sensitive data

## Response expectations

Maintainers should acknowledge reports as soon as practical, validate the issue, and coordinate a fix before public disclosure when possible.
